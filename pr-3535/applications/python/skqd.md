# Sample-Based Krylov Quantum Diagonalization (SKQD)[Â¶](#Sample-Based-Krylov-Quantum-Diagonalization-(SKQD) "Permalink to this heading"){.headerlink}

**Sample-Based Krylov Quantum Diagonalization (SKQD)** is a hybrid
quantum-classical algorithm that combines the theoretical guarantees of
Krylov Quantum Diagonalization (KQD) with the practical efficiency of
sample-based methods. Instead of expensive quantum measurements to
compute Hamiltonian matrix elements, SKQD samples from quantum states to
construct a computational subspace, then diagonalizes the Hamiltonian
within that subspace classically.

::: {#Why-SKQD? .section}
## Why SKQD?[Â¶](#Why-SKQD? "Permalink to this heading"){.headerlink}

Traditional quantum algorithms like VQE face several fundamental
challenges:

-   **Optimization complexity**: VQE requires optimizing many
    variational parameters in a high-dimensional, non-convex landscape

-   **Measurement overhead**: Computing expectation values
    [\\(\\langle\\psi(\\theta)\|H\|\\psi(\\theta)\\rangle\\)]{.math
    .notranslate .nohighlight} requires many measurements for each Pauli
    term

-   **Barren plateaus**: Optimization landscapes can become
    exponentially flat, making gradient-based optimization ineffective

-   **Parameter initialization**: Poor initial parameters can lead to
    local minima far from the global optimum

SKQD addresses these fundamental limitations:

-   âœ… **No optimization required**: Uses deterministic quantum time
    evolution instead of variational circuits

-   âœ… **Provable convergence**: Theoretical guarantees based on the
    Rayleigh-Ritz variational principle

-   âœ… **Measurement efficient**: Only requires computational basis
    measurements (Z-basis), the most natural measurement on quantum
    hardware

-   âœ… **Noise resilient**: Can filter out problematic measurement
    outcomes and handle finite sampling

-   âœ… **Systematic improvement**: Increasing Krylov dimension
    monotonically improves ground state estimates

-   âœ… **Hardware friendly**: Time evolution circuits are more amenable
    to near-term quantum devices than deep variational ansÃ¤tze

::: {#Understanding-Krylov-Subspaces .section}
## Understanding Krylov Subspaces[Â¶](#Understanding-Krylov-Subspaces "Permalink to this heading"){.headerlink}

::: {#What-is-a-Krylov-Subspace? .section}
### What is a Krylov Subspace?[Â¶](#What-is-a-Krylov-Subspace? "Permalink to this heading"){.headerlink}

A **Krylov subspace** [\\(\\mathcal{K}\^r\\)]{.math .notranslate
.nohighlight} of dimension [\\(r\\)]{.math .notranslate .nohighlight} is
the space spanned by vectors obtained by repeatedly applying an operator
[\\(A\\)]{.math .notranslate .nohighlight} to a reference vector
[\\(\|\\psi\\rangle\\)]{.math .notranslate .nohighlight}:

::: {.math .notranslate .nohighlight}
\\\[\\mathcal{K}\^r = \\text{span}\\{\|\\psi\\rangle, A\|\\psi\\rangle,
A\^2\|\\psi\\rangle, \\ldots, A\^{r-1}\|\\psi\\rangle\\}\\\]
:::
:::

::: {#The-SKQD-Algorithm .section}
### The SKQD Algorithm[Â¶](#The-SKQD-Algorithm "Permalink to this heading"){.headerlink}

The key insight of SKQD is that we can: 1. **Generate** Krylov states
[\\(U\^k\|\\psi\\rangle\\)]{.math .notranslate .nohighlight} using
quantum time evolution 2. **Sample** from these states to get
computational basis measurements 3. **Combine** all samples to form a
computational subspace 4. **Diagonalize** the Hamiltonian within this
subspace classically

This approach is much more efficient than computing matrix elements via
quantum measurements!

::: {#Krylov-State-Generation-via-Repeated-Evolution .section}
## Krylov State Generation via Repeated Evolution[Â¶](#Krylov-State-Generation-via-Repeated-Evolution "Permalink to this heading"){.headerlink}

For SKQD, we generate the Krylov sequence:

::: {.math .notranslate .nohighlight}
\\\[\|\\psi_0\\rangle, U\|\\psi_0\\rangle, U\^2\|\\psi_0\\rangle,
\\ldots, U\^{r-1}\|\\psi_0\\rangle\\\]
:::

where [\\(U = e\^{-iH\\Delta t}\\)]{.math .notranslate .nohighlight} is
approximated via Trotter decomposition.

**Implementation Strategy**: 1. Start with reference state
[\\(\|\\psi_0\\rangle\\)]{.math .notranslate .nohighlight} 2. Apply
Trotter-decomposed time evolution [\\(k\\)]{.math .notranslate
.nohighlight} times for [\\(U\^k\|\\psi_0\\rangle\\)]{.math .notranslate
.nohighlight} 3. Measure each Krylov state in computational basis 4.
Accumulate measurement statistics across all Krylov powers

    @cudaq.kernel
    def quantum_krylov_evolution_circuit(
            num_qubits: int, krylov_power: int, trotter_steps: int,
            time_step: float, pauli_operator_words: list[cudaq.pauli_word],
            hamiltonian_coefficients: list[float]):
        """
        CUDA-Q kernel for generating Krylov states via quantum time evolution.

        This kernel implements the core quantum computation of SKQD:
        1. Prepare NÃ©el reference state |ÏˆâŸ© = |010101...âŸ©
        2. Apply time evolution U = e^{-iH*dt} using Trotter decomposition
        3. Repeat k times to get U^k|ÏˆâŸ©
        4. Measure in computational basis
        """

        # Allocate quantum register
        qubits = cudaq.qvector(num_qubits)

        # Step 1: Prepare NÃ©el state as reference |010101...âŸ©
        for qubit_index in range(num_qubits):
            if qubit_index % 2 == 0:
                x(qubits[qubit_index])

        # Step 2: Apply time evolution U^k = (e^{-iH*dt})^k
        for _ in range(krylov_power):  # Apply U exactly k times

            # Each application of U uses Trotter decomposition: U â‰ˆ âˆáµ¢ e^{-ih_i*dt/n}
            for _ in range(trotter_steps):

                # Apply each Pauli term in the Hamiltonian
                # exp_pauli(Î¸, qubits, P) implements e^{-iÎ¸P} for Pauli string P
                for term_index in range(len(hamiltonian_coefficients)):

                    coefficient = hamiltonian_coefficients[term_index]
                    pauli_term = pauli_operator_words[term_index]

                    exp_pauli(coefficient * time_step, qubits, pauli_term)

        # Step 3: Measure all qubits in computational (Z) basis
        mz(qubits)

::: {#Classical-Post-Processing-and-Diagonalization .section}
## Classical Post-Processing and Diagonalization[Â¶](#Classical-Post-Processing-and-Diagonalization "Permalink to this heading"){.headerlink}

Now comes the classical part of SKQD: we use our quantum measurement
data to construct and diagonalize the Hamiltonian within each Krylov
subspace.

1.  **Extract basis states** from measurement counts

2.  **Project Hamiltonian** onto the computational subspace spanned by
    these states

3.  **Diagonalize** the projected Hamiltonian classically

4.  **Extract ground state energy** estimate

::: {#The-SKQD-Algorithm:-Matrix-Construction-Details .section}
### The SKQD Algorithm: Matrix Construction Details[Â¶](#The-SKQD-Algorithm:-Matrix-Construction-Details "Permalink to this heading"){.headerlink}

The core of SKQD is constructing the effective Hamiltonian matrix within
the computational subspace:

1.  **Computational Subspace Formation**: From quantum measurements, we
    obtain a set of computational basis states [\\(\\{\|s_1\\rangle,
    \|s_2\\rangle, \\ldots, \|s_d\\rangle\\}\\)]{.math .notranslate
    .nohighlight} that spans our approximation to the Krylov subspace.

2.  **Matrix Element Computation**: For each Pauli term
    [\\(P_k\\)]{.math .notranslate .nohighlight} in the Hamiltonian with
    coefficient [\\(h_k\\)]{.math .notranslate .nohighlight}:

    ::: {.math .notranslate .nohighlight}
    \\\[H = \\sum_k h_k P_k\\\]
    :::

    We compute matrix elements: [\\(\\langle s_i \| P_k \| s_j
    \\rangle\\)]{.math .notranslate .nohighlight} by applying the Pauli
    string [\\(P_k\\)]{.math .notranslate .nohighlight} to each basis
    state [\\(\|s_j\\rangle\\)]{.math .notranslate .nohighlight}.

3.  **Effective Hamiltonian**: The projected Hamiltonian becomes:

    ::: {.math .notranslate .nohighlight}
    \\\[H\_{\\text{eff}}\[i,j\] = \\sum_k h_k \\langle s_i \| P_k \| s_j
    \\rangle\\\]
    :::

        energy.get() if hasattr(energy, 'get') else energy
