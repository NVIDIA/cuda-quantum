# ADAPT-QAOA algorithm[Â¶](#ADAPT-QAOA-algorithm "Permalink to this heading"){.headerlink}

In this tutorial, we explain the ADAPT-QAOA algorithm introduced in this
[paper](https://journals.aps.org/prresearch/pdf/10.1103/PhysRevResearch.4.033029){.reference
.external} and show how to implement the algorithm in cudaq.

In QAOA (explained in this
[Tutorial](https://nvidia.github.io/cuda-quantum/latest/applications/python/qaoa.html){.reference
.external}), the variational Ansatz consists of [\\(p\\)]{.math
.notranslate .nohighlight} layers, each containing the cost Hamiltonian
[\\(H_C\\)]{.math .notranslate .nohighlight} and a mixer,
[\\(H_M\\)]{.math .notranslate .nohighlight}.

::: {.math .notranslate .nohighlight}
\\\[\\ket{\\psi_p (\\beta, \\gamma)} = \\left ( \\prod\_{k=1} \^p
\[e\^{-i H_M \\beta_k} e\^{-i H_C \\gamma_k}\] \\right )
\\ket{\\psi\_{\\mathrm{ref}}}\\\]
:::

where [\\(\\ket{\\psi\_{\\mathrm{ref}}} = \\ket{+}\^{\\otimes
n}\\)]{.math .notranslate .nohighlight}, [\\(n\\)]{.math .notranslate
.nohighlight} is the number of qubits and [\\(\\gamma, \\beta\\)]{.math
.notranslate .nohighlight} are set of variational parameters. If these
parameters are chosen such that

::: {.math .notranslate .nohighlight}
\\\[\\braket{\\psi_p (\\beta, \\gamma)\| H_C \|\\psi_p (\\beta, \\gamma)
}\\\]
:::

is minimized, then the resulting energy and state provide an approximate
solution to the optimization problem encoded in [\\(H_C\\)]{.math
.notranslate .nohighlight}. In the standard QAOA Ansatz, the mixer is
chosen to be a single-qubit X rotation applied to all qubits.

In ADAPT-QAOA, the single, fixed mixer [\\(H_M\\)]{.math .notranslate
.nohighlight} is replaced by a set of mixers [\\(A_k\\)]{.math
.notranslate .nohighlight} that change from one layer to the next.

::: {.math .notranslate .nohighlight}
\\\[\\ket{\\psi_p (\\beta, \\gamma) } = \\left ( \\prod\_{k=1} \^p
\[e\^{-i A_k \\beta_k} e\^{-i H_C \\gamma_k}\] \\right )
\\ket{\\psi\_{\\mathrm{ref}}}\\\]
:::

The ADAPT-QAOA algorithm can be summarized in three steps:

1- Define the operator set [\\(A_j\\)]{.math .notranslate .nohighlight}
(called the mixer pool, and where [\\(A_j = A\^{\\dagger}\_j\\)]{.math
.notranslate .nohighlight} and select a suitable reference state to be
the initial state: [\\(\\ket{ \\psi (0) }=
\\ket{\\psi\_{\\mathrm{ref}}}\\)]{.math .notranslate .nohighlight};
[\\(\\ket{\\psi\_{\\mathrm{ref}}} = \\ket{+}\^{\\otimes n}\\)]{.math
.notranslate .nohighlight} is chosen as in the standard QAOA

2- Prepare the current Ansatz [\\(\\ket{\\psi(k-1)}\\)]{.math
.notranslate .nohighlight} and measure the energy gradient with respect
to the pool, the [\\(j\\)]{.math .notranslate .nohighlight}th component
of which is given by [\\(âˆ’i \\braket{\\psi(k-1)\|e\^{i H_C \\gamma_k}
\[H_C,A_j\] e\^{âˆ’i H_C \\gamma_k} \|\\psi(k-1)}\\)]{.math .notranslate
.nohighlight}, where the new variational parameter
[\\(\\gamma_k\\)]{.math .notranslate .nohighlight} is set to a
predefined value [\\(\\gamma_0\\)]{.math .notranslate .nohighlight}. For
the measurement, we can decompose the commutator into linear
combinations of Pauli strings and measure the expectation values of the
strings using general variational quantum algorithm methods. If the norm
of the gradient is below a predefined threshold, then the algorithm
stops, and the current state and energy estimate approximate the desired
solution. If the gradient threshold is not met, modify the Ansatz (k) by
adding the operator, A(k) , associated with the largest max component of
the gradient: [\\(\\ket{\\psi(k)}= e\^ {âˆ’i A\_\\mathrm{max} \\beta_k}
e\^{âˆ’i H_C \\gamma_k} \\ket{\\psi(k-1)}\\)]{.math .notranslate
.nohighlight}, where [\\(\\beta_k\\)]{.math .notranslate .nohighlight}
is a new variational parameter

3- Optimize all parameters currently in the Ansatz [\\(\\beta_m,
\\gamma_m = 1, 2, \...k\\)]{.math .notranslate .nohighlight} such that
[\\(\\braket{\\psi (k)\|H_C\|\\psi(k)}\\)]{.math .notranslate
.nohighlight} is minimized, and return to the second step.

    def spin_ham (edges, weight):

        ham = 0

        count = 0
        for edge in range(len(edges)):

            qubitu = edges[edge][0]
            qubitv = edges[edge][1]
            # Add a term to the Hamiltonian for the edge (u,v)
            ham += 0.5 * (weight[count] * cudaq.spin.z(qubitu) * cudaq.spin.z(qubitv) -
                                weight[count] * cudaq.spin.i(qubitu) * cudaq.spin.i(qubitv))
            count += 1

        return ham

    # Collect coefficients from a spin operator so we can pass them to a kernel

        result = []

    def qaoa_mixer(n):

        term = spin.x(0)

        for i in range(1, n):
            term = term + spin.x(i)

        pool = [term]
        return pool

    def qaoa_single_x(n):

        pool = []

        for i in range(n):
            pool.append(cudaq.SpinOperator(spin.x(i)))

        return pool

    def qaoa_double_ops(n):

        pool = []

        for i in range(n-1):
            for j in range(i+1, n):
                pool.append(cudaq.SpinOperator(spin.x(i)) * cudaq.SpinOperator(spin.x(j)))
                pool.append(cudaq.SpinOperator(spin.y(i)) * cudaq.SpinOperator(spin.y(j)))
                pool.append(cudaq.SpinOperator(spin.y(i)) * cudaq.SpinOperator(spin.z(j)))
                pool.append(cudaq.SpinOperator(spin.z(i)) * cudaq.SpinOperator(spin.y(j)))

        return pool

    def mixer_pool(qubits_num):
        return (
              qaoa_single_x(qubits_num)
            + qaoa_mixer(qubits_num)
            + qaoa_double_ops(qubits_num)
