Python API — cudaq.ptsbe¶

Sampling Functions¶

cudaq.ptsbe.sample(kernel, *args, shots_count=1000, noise_model=None, max_trajectories=None, sampling_strategy=None, shot_allocation=None, return_execution_data=False)¶

Sample a quantum kernel using Pre-Trajectory Sampling with Batch Execution.

Pre-samples T unique noise trajectories from the circuit’s noise model and batches circuit executions by unique trajectory. Each trajectory is simulated as a pure-state circuit; results are merged into a single SampleResult.

When any argument is a list (broadcast mode), the kernel is executed for each element of the list and a list of results is returned.

Parameters:

  • kernel – The quantum kernel to execute. Must be a static circuit with no mid-circuit measurements or measurement-dependent conditional logic.

  • args – Positional arguments forwarded to the kernel.

  • shots_count (int) – Total number of measurement shots to distribute across all trajectories. Default: 1000.

  • noise_model (cudaq.NoiseModel or None) – Noise model describing gate-level error channels. Noise can also be injected inside the kernel via cudaq.apply_noise(); both can be combined. Default: None (no noise).

  • max_trajectories (int or None) – Maximum number of unique trajectories to generate. None defaults to shots_count. Setting an explicit limit (e.g. 500) enables trajectory reuse and is strongly recommended for large shot counts.

  • sampling_strategy (PTSSamplingStrategy or None) – Strategy used to select trajectories from the noise space. None uses the default ProbabilisticSamplingStrategy.

  • shot_allocation (ShotAllocationStrategy or None) – Strategy used to distribute shots across the selected trajectories. None uses PROPORTIONAL.

  • return_execution_data (bool) – When True, attaches the full execution trace (circuit instructions, trajectory specifications, and per-trajectory measurement counts) to the returned result. Default: False.

Returns:

Aggregated measurement outcomes. In broadcast mode, a list of results is returned.

Return type:

cudaq.ptsbe.PTSBESampleResult

Raises:

RuntimeError – If the kernel contains mid-circuit measurements, conditional feedback, unsupported noise channels, or invalid arguments.

import cudaq
from cudaq import ptsbe

cudaq.set_target("nvidia")

@cudaq.kernel
def bell():
    q = cudaq.qvector(2)
    h(q[0])
    cx(q[0], q[1])
    mz(q)

noise = cudaq.NoiseModel()
noise.add_channel("h", [0], cudaq.DepolarizationChannel(0.01))

result = ptsbe.sample(bell, shots_count=10_000,
                      noise_model=noise, max_trajectories=200)
print(result)
cudaq.ptsbe.sample_async(kernel, *args, shots_count=1000, noise_model=None, max_trajectories=None, sampling_strategy=None, shot_allocation=None, return_execution_data=False)¶

Asynchronous variant of sample(). Submits the job without blocking and returns a future.

All parameters are identical to sample().

Returns:

A future whose .get() method returns the PTSBESampleResult.

Return type:

AsyncSampleResult

future = ptsbe.sample_async(bell, shots_count=10_000, noise_model=noise)
# ... do other work ...
result = future.get()

Result Type¶

class cudaq.ptsbe.PTSBESampleResult¶

Extends cudaq.SampleResult with an optional PTSBEExecutionData payload produced when return_execution_data=True.

has_execution_data() → bool¶

Return True if execution data is attached to this result.

execution_data() → PTSBEExecutionData¶

Return the attached PTSBEExecutionData.

Raises:

RuntimeError – If no execution data is available. Check has_execution_data() first.

Trajectory Sampling Strategies¶

class cudaq.ptsbe.PTSSamplingStrategy¶

Abstract base class for trajectory sampling strategies. Subclass and implement generate_trajectories() to define a custom strategy.

generate_trajectories(noise_points, max_trajectories) → list[KrausTrajectory]¶

Generate up to max_trajectories unique trajectories from the given noise points.

Parameters:

  • noise_points – Noise site information extracted from the circuit.

  • max_trajectories (int) – Upper bound on the number of trajectories.

Returns:

List of unique KrausTrajectory objects.

name() → str¶

Return a human-readable name for this strategy.

class cudaq.ptsbe.ProbabilisticSamplingStrategy(seed=0)¶

Randomly samples unique trajectories weighted by their occurrence probability. Produces a representative cross-section of the noise space. Duplicate trajectories are discarded.

Parameters:

seed (int) – Random seed for reproducibility. 0 uses the global CUDA-Q seed if set, otherwise a random device seed.

strategy = ptsbe.ProbabilisticSamplingStrategy(seed=42)
result = ptsbe.sample(bell, shots_count=10_000,
                      noise_model=noise,
                      sampling_strategy=strategy)
class cudaq.ptsbe.OrderedSamplingStrategy¶

Selects the top-T trajectories sorted by probability in descending order. Ensures the highest-probability noise realizations are always represented. Best when the noise space is dominated by a small number of likely error patterns.

result = ptsbe.sample(bell, shots_count=10_000,
                      noise_model=noise,
                      max_trajectories=100,
                      sampling_strategy=ptsbe.OrderedSamplingStrategy())
class cudaq.ptsbe.ExhaustiveSamplingStrategy¶

Enumerates every possible trajectory in lexicographic order. Produces a complete representation of the noise space. Only practical when the noise space is small (few noise sites and low Kraus operator count).

class cudaq.ptsbe.ConditionalSamplingStrategy(predicate, seed=0)¶

Samples trajectories that satisfy a user-supplied predicate function. Useful for targeted studies such as restricting to single-qubit error events or trajectories below a probability threshold.

Parameters:

  • predicate – A callable (KrausTrajectory) -> bool that returns True for trajectories to include.

  • seed (int) – Random seed. 0 uses the global CUDA-Q seed.

# Keep only trajectories with at most one error
strategy = ptsbe.ConditionalSamplingStrategy(
    predicate=lambda traj: traj.count_errors() <= 1,
    seed=42,
)
result = ptsbe.sample(bell, shots_count=10_000,
                      noise_model=noise,
                      sampling_strategy=strategy)

Shot Allocation Strategy¶

class cudaq.ptsbe.ShotAllocationStrategy(type=ShotAllocationStrategy.Type.PROPORTIONAL, bias_strength=2.0, seed=0)¶

Controls how the total shot count is distributed across the selected trajectories after trajectory sampling.

Parameters:

  • type (Type) – Allocation strategy type.

  • bias_strength (float) – Exponent used by the biased strategies. Higher values produce stronger bias. Default: 2.0.

  • seed (int) – Random seed used by probabilistic allocation (PROPORTIONAL and biased strategies). 0 uses the global CUDA-Q seed.

class Type¶
PROPORTIONAL¶

(default) Shots are allocated via multinomial sampling weighted by trajectory probability. The total is always exactly shots_count and every trajectory with non-zero probability receives a fair share.

UNIFORM¶

Equal shots per trajectory regardless of probability.

LOW_WEIGHT_BIAS¶

Biases more shots toward trajectories with fewer errors (lower Kraus weight). Weight formula: (1 + error_count)^(-bias_strength) * probability.

HIGH_WEIGHT_BIAS¶

Biases more shots toward trajectories with more errors. Weight formula: (1 + error_count)^(+bias_strength) * probability.

alloc = ptsbe.ShotAllocationStrategy(
    ptsbe.ShotAllocationStrategy.Type.LOW_WEIGHT_BIAS,
    bias_strength=3.0,
)
result = ptsbe.sample(bell, shots_count=10_000,
                      noise_model=noise,
                      shot_allocation=alloc)
